Option A — Deploy to Vercel

Recommended · ~12 min
ℹ️ You need Node.js 18+ installed. Check with node -v. Download from nodejs.org if needed.
1
Scaffold a new Vite + React project
Open your terminal and run these commands. Vite is the build tool that compiles your React JSX into deployable HTML/CSS/JS.
Terminal
npm create vite@latest isms-portal -- --template react
cd isms-portal
npm install
2
Replace the App file with your ISMS portal
Copy your downloaded ISMS-portal.jsx file into the project and update the entry point.
File actions
# Copy ISMS-portal.jsx into src/
cp ~/Downloads/ISMS-portal.jsx src/App.jsx
Then open src/main.jsx and make sure it looks like this:
src/main.jsx
import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App.jsx'

ReactDOM.createRoot(document.getElementById('root')).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
)
3
Test locally
Run the development server and confirm everything looks correct at http://localhost:5173 before deploying.
Terminal
npm run dev
✅ If the portal loads and all pages work — you're ready to deploy. Press Ctrl+C to stop the dev server.
4
Install Vercel CLI and deploy
Install the Vercel CLI globally, then run vercel from your project folder. It will walk you through login and deployment.
Terminal
npm install -g vercel
vercel
Answer the prompts:
? Set up and deploy?          → Y
? Which scope?                  → Your username
? Link to existing project?     → N
? Project name?                 → isms-portal
? Directory?                    → ./  (press Enter)
? Override build command?       → N  (press Enter)
? Override output dir?          → N  (press Enter)
🚀 Vercel will print your live URL — something like https://isms-portal-xyz.vercel.app. That's your portal, live!
5
Fix the 404 on page refresh (SPA routing)
React Router apps need a redirect rule so that refreshing any page doesn't return a 404. Create a vercel.json file in the project root:
vercel.json (project root)
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
Then redeploy:
Terminal
vercel --prod
6
(Optional) Add a custom domain
In the Vercel dashboard → your project → Settings → Domains — add any domain you own. Vercel auto-provisions SSL. For future updates, just run vercel --prod again from your project folder.
💡 Auto-deploy tip: Push your project to GitHub, link the repo in Vercel, and every git push will auto-deploy. Zero extra commands needed.

Option B — Deploy to Netlify

Easiest drag-and-drop · ~8 min
1
Build your project
Follow Steps 1–3 from the Vercel guide to scaffold and set up the project, then run the build command:
Terminal
npm run build
This creates a dist/ folder containing your compiled app ready for deployment.
2
Fix SPA routing before deploying
Create a _redirects file inside your public/ folder (Vite copies it to dist/ automatically):
public/_redirects
/*    /index.html   200
Then rebuild:
npm run build
3
Drag and drop to deploy
Go to app.netlify.com → sign up free → scroll to the bottom of the dashboard where it says "Want to deploy a new site without connecting to Git?" → drag your entire dist/ folder onto that zone.
✅ Your site goes live instantly with a URL like https://graceful-swan-abc123.netlify.app. Rename it under Site settings → General.

Option C — GitHub Pages

For developers · ~20 min
1
Configure Vite base path
Open vite.config.js and set the base path to match your GitHub repo name:
vite.config.js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  base: '/isms-portal/'  // ← your GitHub repo name
})
2
Install gh-pages and configure package.json
Terminal
npm install --save-dev gh-pages
Add these to package.json:
package.json (add these lines)
{
  "homepage": "https://YOUR-USERNAME.github.io/isms-portal",
  "scripts": {
    "predeploy": "npm run build",
    "deploy": "gh-pages -d dist"
  }
}
3
Push to GitHub and deploy
Terminal
# Create repo on github.com first, then:
git init
git add .
git commit -m "Initial ISMS portal"
git remote add origin https://github.com/YOUR-USERNAME/isms-portal.git
git push -u origin main

# Deploy to GitHub Pages
npm run deploy
Your portal will be live at https://YOUR-USERNAME.github.io/isms-portal within 2–3 minutes.
